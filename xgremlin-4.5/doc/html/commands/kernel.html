kernel

Syntax: kernel newnop

The kernel function is assumed to be in the r array. It is masked, stretched to newnop points, and transformed; the transform is phased so that the center of gravity is at nop/2 + 1. This command was designed to permit matching the length of an instrumental profile scan to the scans which are to be deconvolved using the instrumental profile, but may be used for any kernel function to be used for convolution or restoration.

Note:
Unlike in MS/DOS - Gremlin the resulting kernel is not automatically saved in the ffta array because there may be no ffta array allocated when the kernel command is typed in.
