interpolate

Syntax:

  • interpolate disp wstart wstop
  • interpolate disp wstart np
  • interpolate disp ni np
  • interpolate disp ni
  • interpolate disp
Parameters:
disp
dispersion in wavenumber/wavelength per point
wstart
wavenumber/wavelength to start interpolation
wstop
wavenumber/wavelength to finish interpolation
ni
point subscript to start interpolation
np
number of points to interpolate
The current buffer (contents of r) is interpolated onto the wavenumber/wavelength scale specified by disp (which is given in wavenumbers or nm per point), starting at wavenumber/wavelength wstart or point ni and continuing until wavenumber/wavelength wstop, or until np points have been interpolated; the result replaces the original data in r. It is not possible to obtain more than nrtr (number of points in r and tr arrays) points although the call may specify parameters which would require more than nrtr points. If a call requests an interpolation through wavenumbers/wavelengths outside the range of the data in the r array, as many points as possible are interpolated (the routine will try to do the best it can with the situation). The information block entries nop, wref and delw are all modified by this command.

Note:
If disp is negative, it is interpreted as a step size in the alternate (wavelength/wavenumber) regime; for wstart, use the highest value wanted, and for wstop use the lowest. The contents of the tr array are destroyed. The tr array is used as a scratch work area by this command.
