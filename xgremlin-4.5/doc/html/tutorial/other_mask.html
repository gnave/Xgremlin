Masking data before transforming

Sometimes you may want to do a Fourier transform of data in the r array even if the number of data points is not a power of two. The solution is to padd the interferogram up to a power of two with zeros, but this has the disadvantage of leaving a large jump in the interferogram, which gives bad ringing in the resulting spectrum. The solution is to multiply the ends of the data with a cosine bell before padding with zeros, thus bringing the data smoothly down to zero. This is done with the command mask nrise mean where nrise is the width in points of the cosine bell. A suitable value is 5-10 %. If you specify the mean as 0., the mask command will set the mean of the interferogram equal to zero before masking it (otherwise you might still get an ugly jump if your data is not centred around zero).

You can then pad the interferogram with zero to a power of two with the zeropad command before transforming it with fourtran.
