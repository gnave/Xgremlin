save

Syntax:

save <key> [<npts>] [tr] [<new id>]

  • <key> : an integer 1-29, except 17,18,19 to specify the scratch file
  • <npts>: the number of points to be saved to the scratch file

Saves data from the 'r' or 'tr' arrays into a scratch file. If the number of points to be saved is not specified all points (nop) will be saved. By default data in the 'r' array will be saved.

Example:

save 1 "nice spectrum"
will store all points in the 'r' array in a file scratch.1 in the current scratch file directory. 
save 2 tr 1024
will save 1024 points from the 'tr' array in a file named scratch.2
