button

Syntax:

  • button < number > edit < "button label" > < "command string" >
  • button < number > plot < "button label" > < "plot command" >
The button command is used to assign a command to one of the buttons between the ploting and editing window. Buttons are numbered from 1 to 8 (left to right). There are two types of commands: edit commands are those that would normally be typed in at the prompt in the text window of Xgremlin (e.g. dispar). To assign an edit command to a button the following form of the button command must be used:

button < number > edit < "button label" > < "command string" >

Example: button 1 edit sinc "sinc 10. 500;plot"

will label the leftmost button sinc and assign the command 'sinc 10. 500;plot' to the button (calculate a sinc function in the r-array and plot the result on the screen). (It is also possible to assign some useful command to a button.)

For plot commands the command has the same form as described in help topic Plot mode commands:
< modifier > + < key >

Example 1: button 2 plot Zoom-in x

This labels the 2nd button from the left with 'Zoom-in' and assigns to it the command 'x' which lets you zoom in at the center of the plot (or at a set marker). None of the keys (or ) is pressed.

Example 2: button 4 plot Zoom-out M+x

assigns the plot command < meta > -x (or < alt > -x on a PC) used for zooming out to button number 4.

Note:
Following the usual Xgremlin conventions, strings containing blank spaces such as in "sinc 10. 500" must be enclosed in quotes so they are recognised as a single string by Xgremlin.

For a list of plot commands see help topic Plot mode commands.
