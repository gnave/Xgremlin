writelines

Syntax:

writelines <filename> [ <wstart> <wstop> ] [ <smin> ] [ <smax> ] [intcorr] [aircorr] [wavcorr]

where

  • wstart - first wavenumber wanted
  • wstop - last wavenumber wanted (Default is all lines in the file)
  • smin - minimum line strength written to file
  • smax - maximum line strength written to file
Writelines reads the line parameters from the .lin file attached to unit lineio in the desired range and writes them out to file "filename" as an ASCII file. This file may be sent to the terminal instead by simply using "con" as the filename.

If an upper limit (smax) is wanted, all four parameters must be given.

If intensity correction is wanted have the (lo-res) response function of the optical system in r and add the command parameter intcorr. See the description of the 'response' command for a way to obtain the response function. if the response function is zero, lines at these wavenumbers are not written out to the linelist file.

If the command parameter aircorr is added on the command line the wavenumbers in the line list are corrected for the refractive index of air (Birch and Downs formulae). The air correction is not applied by default. Pressure and temperature parameters for the air correction are taken from the file header.

The command parameter wavcorr requests that the linear wave number correction is applied to the wavenumbers in the line list. The wavenumber correction is not applied by default.

The held parameters (H) are coded as follows:

  • 1 = damping
  • 2 = width
  • 4 = sigma
  • 8 = amplitude
If more than one parameter is held, H = the sum of the individual values. System tags (4th tag character) have the following meaning:
  • I = line was manually inserted
  • F = line came from a derivative findlines or lineseek
  • L = line has been through lsqfit
  • R = line has been through a filter lsqfit ("ringfit")
A description of the output format can be found in the Xgremlin tutorial in the section on finding and fitting lines.

Example:

writelines neon.data intcorr
will read line data records from the line list file that is currently attached to the 'lineio' unit and write the line list data to the ASCII file "neon.data" after after dividing intensities by the optical response function in the 'r' array.
