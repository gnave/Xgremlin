open

Syntax: open < unit name > < file name > [ new ]

Xgremlin is mostly written in Fortran and retains a certain Fortran-ish feel. One of the surviving Fortran anachronisms in Xgremlin is that certain named and unalterable units are used for reading and writing data from and to files. Before reading from or writing to a file, a file name has to be assigned to a suitable unit. Subsequently the file is then referred to by the unit name.

The Xgremlin unit names are:

com
The batch file (cards file) unit. Does not normally need to be specifically opened.

datain
The normal unit for reading in spectral data. (not interferograms).

dataout
For binary output files. Usually opened using create .

lineio
For files with binary line list data (.lin files). If the .lin file does not exist and new is specified on the command line, a new .lin file will be created.

altdata
An alternative datain unit.

rawin
for interferograms to be transformed.

scratch
For saving any data temporarily in the scratch directory. The scratch directory lives in a users home directory or the directory pointed to by the XGREMLIN_SCRATCH_PATH environment variable.

spool
A file to take output usually directed to the screen (see spool for use).

logfile
To log all the commands you type in. (default name: xgremlin-(date)-(time).log in the home directory).


Examples

open datain k11sh2
will assign the file k11sh2.dat to the input unit datain. Subsequent read commands will then read data from the file k11sh2.dat

open lineio k11_a
will assign the file k11_a.lin to the unit lineio. Subsequent getlines and putlines commands will then read line list data from (or write them to) the file k11_a.lin

open lineio foobar new
will create a new linelist file foobar.lin and assign it to the unit lineio. This form of the command allows to write internal line lists to a file after creating them e.g. with a findlines command.
