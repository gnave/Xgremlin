printer

Syntax:

printer <name>

sets the name of the printer which is to be used for screen dumps. The printer name can be the name of any local or remote printer defined in the printer database /etc/printcap. If the printer name is set to 'none' only the screen dump in Postscript format will be written to disk but it will not be sent to the printer.

NOTE 1:

Xgremlin screen dumps are in colour Postscript format. They will print on a 'normal' Postscript printer. If no Postscript printer is available the free Postscript interpreter 'Ghostscript' can be used to print the screen dump on any printer. See the file HOWTO-INSTALL' which is part of the Xgremlin source distribution for information about obtaining and installing 'Ghostscript'.

NOTE 2:

The name of the Postscript files written to disk by Xgremlin during a screen dump is: xgremlin-<pid>.ps where <pid> is the process id of the running Xgremlin. The pid is used in the file name to make screen dump files unique to prevent overwriting of screen dumps in case several instances of Xgremlin are running.

Examples:

printer stylus-ps
screen dumps will be printed to a printer named 'stylus-ps' in the printer database /etc/printcap 
printer none
suppresses screen dumps. The plot is still recorded in the file xgremlin-<pid>.ps, and can be viewed or sent to the printer separately.
