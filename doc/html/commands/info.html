info

Syntax: info < variable name > [ < new value > ]

Xgremlin uses a number of control variables whose current value can be displayed and (optionally) modified with the info command. The info command typed in without parameters alone prints the entire information block. Some control variables such as delw will be reset every time a new data file is opened with a read command.

Note:
Some of the functionality of the info command is duplicated in other commands. See the section related commands.

The meanings of the control variables are:

bocode
byte order code. 0=little endian; 1=big endian
clkcnt
unknown
day
date data taken.
delw
dispersion (number of wavenumbers per point)
fboffs
file byte offset - number of bytes to skip at beginning.
ftsmode
unknown
fzeeman
unknown
id
identification of file (at top of header file)
inftype
type of header file (newest is type 6)
natod
unknown
neoa
neoa=1 for emission, 0 for absorption spectra
nint
unknown
nk
divisions/fringe
nm
divisions/sample
noiselev
noise level (where is this used?)
nop
number of points used in current buffer
noptex
unknown
npo
total number of points in file (??)
ns
carriage travel parameter
nwos
=-1 for wavenumbers; =1 for wavelengths
nwpp
nwpp=1 for real spectra, 2 for complex spectra
pid
print process id of running Xgremlin
pspect
spectrometer pressure in torr.
refwavno
wavenumber of reference wavenumber (15798.0025 for HeNe)
rdsclfct
real scale factor (data multiplied by rdsclfct on read).
resolutn
resolution
sample
sample
user
user
uvar0
user variable 0 (e.g. for standards)
uvar1
user variable 1 (e.g. for standards)
wavcorr
wavenumber correction factor (used in writelines ).
wstart
first wavenumber of file
wstop
last wavenumber of file
wref
first wavenumber or reference wavenumber in current buffer
size
size of available data buffers in Xgremlin (cannot be changed)
vacuum
if True use vacuum wavelengths, if False use air wavelengths for writelines .
margin
size of top/bottom margin of plotter window in % (an integer value)
stol
Controls the spline phase fit. Must be between 0 and 1. If stol = 0, the spline is an interpolating spline (goes through the points). Values of stol larger than 0 describe how closely the approximating spline approaches the data points.
sten
Controls the spline phase fit. The tension factor must be positive. If sten = 0, the spline is a normal cubic spline.
smof
Controls the spline phase fit. The larger this factor, the larger the degree of smoothing.


Examples:

info nwpp 2
tells Xgremlin to treat data in the r array as complex.

info delw 1.0
set the dispersion to 1.0 wavenumbers / point

info vacuum true
uses vacuum wavelengths instead of air wavelengths in writelines .
