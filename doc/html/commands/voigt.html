voigt

Syntax:

  • voigt <w> <nop>
  • voigt <w> <nop> <ncenter>
  • voigt <wl> <wg> <nop>
where
  • w, wl - FWHM of the Lorentzian function (points)
  • wg - FWHM of the Gaussian function (points)
  • ncenter - central point of Lorentzian
  • nop - number of points in the generated function
This command generates a function in the 'r' array, covering a range of NOP points. The first two forms generate a Lorentzian function, given by:

In the first form ncenter = nop / 2 + 1.

The three parameter form can generate in 'r', a Lorentzian profile, a Gaussian profile or a convolution of the two. The two parameters wl and wg govern the contribution of each type of the function to the final profile. The simple cases occur when either wl or wg is zero. If wl is zero, the resulting function will be a Gaussian, and vice versa. This form uses an inverse Fourier transform to generate the Voigt profile. For this reason nop must be large enough for the function to drop off to zero within its bounds, to avoid aliasing effects. This form also destroys the previous contents of 'tr'. The resulting Voigt profile can be moved to the desired position with the command shift.

Examples:

voigt 0. 10. 512
makes a Gaussian profile of width 10 points, centred at 256 points, in an r array of length 512 points. 
voigt 10. 200 512
generates a Lorentzian of width 10 points, centred at 200 points and 512 points wide.
