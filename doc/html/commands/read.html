read

Syntax:

  • read <wstart>
  • read <wstart> [ <wstop> ] [ <unit name> ]
  • read <wstart> [ <npts> ] [ <unit name> ]
  • read "offset" <noffset>
where
  • unit name" - unit name to which the unit number is assigned
  • wstart - starting wavenumber(length)
  • wstop - stopping wavenumber(length)
  • npts - the number of points to read; max of 8192, default 2048 (see set readlength)
Reads in a block of data from the unit unit name, starting at wstart and ending either at wstop, or after npts data points have been read in.

A read with no unit specified defaults to the datain unit. Other valid names are:

  • datain
  • dataout
  • altdata.
The data will always be scaled by the factor rdsclfct which is defined in the header file during reading. The scale factor may also be set with the command set "read scale". This scaling is undone with set "read noscale". In header files produced by Xgremlin rdsclfct is always set to 1.

The data can be normalized such that the maximum in the r array is always set to 1 using the command set "read normalize". This undoes the standard input scaling, and can be reversed with the command set nonorm.

The last command sets a flag, but does not actually read.

read offset <noffset>
ignores the first <noffset> bytes before calculating which block of data to read in; this can be useful when the data is preceded by a header block.

Examples

read 33000. 65536
reads in 65536 points starting at the wavenumber 33000. 
read 33000. 35000. altdata
reads in a block of data between 33000. and 36000. wavenumbers from the file attached to the unit "altdata"
