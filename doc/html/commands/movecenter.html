movecenter

Syntax: movecenter npoints

This can be done during interactive phase correction to find the correct centre to use - i.e. the one that gives the best phase across the spectrum. The phase from an incorrect centre will not be particularly flat, and (more seriously) may have 2*pi phase flips. Change the central point ncenter by npoints and re-run input center.

Note:
this is more easily done using the Phase / Transform form in the Gremlins menu.
