readdata

Syntax:

  • readdata [ (<format>) ] [ <nop> ]
  • readdata (<format>) [ <unit-name> ] [ <nop> ]

where

  • "format" - any valid FORTRAN format specification for REAL numbers. (must include parentheses)
  • "unit-name" - one of the symbolic Xgremlin unit names
  • nop - the number of points to be read in.
Readdata will read the format specifier as the 1st line of the input if not given on the command line. By default the input will be taken from the source of command input unless overridden. The number of points may be set prior to issuing this command or on the command line. The unit which is read may be reset using the "unit-name"; appropriate Xgremlin unit-names are:
  • com
  • outfile
readdata is most commonly used to read in ASCII data from a file, especially that which has been produced with writefile.

Examples:

The follwing commands in a file will read the data in the specified format into the r array: 
info wref      1199.16334
info id 'Some meaningful title                                            '
info delw     .15585311200E-02
length  90000
