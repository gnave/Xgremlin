alloc

Syntax: alloc [< array > < size >]

used to resize the r, tr, fft and line list arrays dynamically. The r and tr arrays are always resized together. Since the fft arrays is used solely for Fourier transforms the size is always rounded up to the nearest power of 2. When the program is started up the size of the fft arrays is only 256k points to reduce the memory consumption of the program.

It is also possible to use the shorthand notation k for 1024 and m for 1024*1024 e.g. 256k may be typed instead of 262144.

When the alloc command is typed without parameters the current size of the arrays is printed out. The info size command can also be used to print out the current size of the internal buffers in Xgremlin.


Examples:

alloc r 500000
creates r and tr arrays for 1/2 million points.

alloc fft 1000000
creates a Fourier transform array for 1048576 points.

alloc lines 2000
creates an internal line list that can hold data of 2000 spectral lines.

alloc r 256k
creates r and tr arrays for 256*1024 points
